== Core principles

=== Local-first

Launchpad runs entirely on your machine. Source code, scan results, and generated context never leave the local filesystem. The local AI model runs through Ollama or a compatible local endpoint -- no cloud API calls are made during analysis or synthesis.

=== Deterministic-first

Repository structure, dependencies, endpoints, and documentation are extracted through structured, deterministic analysis. The local AI model is only used for bounded synthesis -- writing short narrative sections from data the deterministic phase already produced.

This ordering is deliberate. Deterministic analysis is fast, repeatable, and does not hallucinate. AI synthesis fills in the parts that benefit from natural language -- project overview, architectural notes -- but it operates on structured inputs, not raw source code.

=== Vendor-neutral output

Generated context files use a standard shape (`AGENTS.md` + `.ai/`) that any AI assistant can read. Agent-specific formats (Claude Code skills, Cursor rules, Windsurf rules) are produced as optional projections on top of the canonical output, not as replacements for it.

Alongside the prose output, two deterministic machine-readable sidecars are emitted: a graph projection of the project structure (`.launchpad/project.model.json`, see xref:project-model.adoc[the project-model sidecar]) and a per-rule projection of the resolved standards (`.launchpad/standards.index.json`, see xref:standards-index.adoc[the standards-index sidecar]). Both let an agent query structured context instead of parsing prose.
