= Output contract

The integration contract for everything Launchpad writes into a target
repository. It pins down the file layout, the section structure of each
generated artifact, the machine-readable provenance stamp, the write/merge
semantics, and - crucially - which parts are *stable* (safe to integrate
against) versus *experimental* (wording may change run-to-run). Downstream
tools (Claude Code, Cursor, Windsurf, the MCP server) can build on this instead
of snapshot-testing generated prose.

== Why

A scan emits a tree of files into the project. Consumers need to know what those
files are called, where they live, how their sections are shaped, and what
signals a breaking change - without diffing prose between releases. This
document is that promise. The deterministic sidecars under `.launchpad/` each
carry their own `schemaVersion` and are documented separately
(xref:project-model.adoc[], xref:standards-index.adoc[]); this page is the
contract for the *human/agent-facing* surface (`AGENTS.md`, `.ai/*`) and the
vendor projections, plus the rules that govern all of them.

== File layout

A scan writes into the project root. Each line is tagged with its stability
tier (see <<Stability tiers>>).

[source]
----
<project-root>/
├── AGENTS.md                       # primary agent-instructions file        [stable layout / experimental prose]
├── .ai/
│   ├── index.md                    # navigation index + skills list         [stable]
│   ├── stack.md                    # detected stack and dependencies        [stable]
│   ├── engineering-rules.md        # rules (only if rules exist)            [stable]
│   ├── skills.md                   # skills (only if skills exist)          [stable]
│   └── checklists.md               # checklists (only if checklists exist)  [stable]
├── .claude/
│   └── skills/<skill-id>/SKILL.md  # Claude Code projection (default-on)     [stable]
├── .cursor/
│   └── rules/*.mdc                 # Cursor projection (opt-in)              [stable]
├── .windsurf/
│   └── rules/*.md                  # Windsurf projection (opt-in)            [stable]
└── .launchpad/
    ├── project.model.json          # deterministic structure graph          [stable, versioned]
    ├── standards.index.json        # deterministic standards records        [stable, versioned]
    ├── audit.md                    # audit findings, human-readable         [stable]
    ├── audit.sarif.json            # audit findings, SARIF 2.1.0            [stable]
    └── backups/<timestamp>/...     # pre-write backups                       [internal]
----

Conditional companions (`engineering-rules.md`, `skills.md`, `checklists.md`)
are written only when the resolved standards pack contributes that record type.
The projection directories appear only for enabled projections - `claude` is
on by default; `cursor` and `windsurf` are opt-in via the standards manifest's
`projections` list.

== Generated files

[cols="2,1,2,1"]
|===
| File | Kind | Produced by | Conditional

| `AGENTS.md`              | `CONTEXT` | deterministic + AI-synthesized intro | no
| `.ai/index.md`          | `INDEX`   | deterministic | no
| `.ai/stack.md`          | `CONTEXT` | deterministic | no
| `.ai/engineering-rules.md` | `RULES` | deterministic (from pack) | only if rules exist
| `.ai/skills.md`         | `SKILLS`  | deterministic (from pack) | only if skills exist
| `.ai/checklists.md`     | `CONTEXT` | deterministic (from pack) | only if checklists exist
| `.claude/skills/<id>/SKILL.md` | `SKILL` | deterministic projection | per skill (default-on)
| `.cursor/rules/*.mdc`   | `RULES` / `SKILL` | deterministic projection | opt-in
| `.windsurf/rules/*.md`  | `RULES` / `SKILL` | deterministic projection | opt-in
|===

`Kind` is the stable `FileKind` discriminator carried on every generated file
(`CONTEXT`, `RULES`, `SKILL`, `SKILLS`, `INDEX`). "Deterministic" means the
content is assembled from the scan and standards pack with no model call;
"AI-synthesized" means a local model authored the prose (with a deterministic
fallback when synthesis is disabled).

=== AGENTS.md sections

`AGENTS.md` is assembled in a fixed order. Sections render only when the scan
produced the underlying signal; the order itself is stable.

[cols="2,1,2"]
|===
| Section | Conditional | Source

| H1 title               | always | project name (deterministic)
| Intro                  | always | *AI-synthesized* (deterministic fallback)
| Commands               | if a build tool is detected | deterministic
| Architecture           | if sources are classified | deterministic
| Endpoints              | if HTTP routes are found | deterministic
| Workflows              | if workflows are modeled | deterministic
| Operations             | if operations are detected | deterministic
| Risks                  | if risks are inferred | deterministic
| Inferred Standards     | if standards are inferred | deterministic
| Companion Pointers     | if companions were written | deterministic
| Boundaries             | always | deterministic
|===

The managed body of `AGENTS.md` lives between merge markers (see
<<Write semantics and merge markers>>); the provenance stamp sits inside that
block so a re-run refreshes it in place.

== Provenance header

Every generated context file (`AGENTS.md` and each `.ai/*.md` companion) opens
with a single-line, machine-readable lineage stamp: a stable marker token
followed by a compact JSON payload, rendered as an HTML comment so every
markdown renderer tolerates it.

[source]
----
<!-- launchpad:provenance {"schemaVersion":1,"launchpadVersion":"0.6.0","generatedAt":"2026-06-22T14:30:45Z","standards":{"pack":"acme-standards","version":"1.2.0","origin":"remote-cache"},"aiModel":"qwen2.5-coder"} -->
----

[cols="1,3"]
|===
| `schemaVersion`    | Integer. Bumped on any breaking change to this payload shape. Currently `1`.
| `launchpadVersion` | The running Launchpad build version, or `dev` for IDE runs.
| `generatedAt`      | ISO-8601 generation timestamp. Excluded from every content hash.
| `standards`        | Resolved standards-pack provenance, or JSON `null` when no pack resolved. Object of `pack`, `version`, `origin` (e.g. `remote-cache`, `project-local`).
| `aiModel`          | The model that synthesized prose, or `deterministic-only` when no model ran.
|===

The `launchpad:provenance` marker token is stable and greppable; the JSON
payload is parseable. A reader can answer "what produced this?" without
consulting git history.

== Write semantics and merge markers

Launchpad-managed content is wrapped in sentinel comments so a file may be
hand-edited around the managed block and still refreshed safely on re-run.

[source]
----
<!-- launchpad:managed:start -->
...managed content (replaced in place on every re-run)...
<!-- launchpad:managed:end -->
----

On each run the target path is classified into a write action. Integrators can
rely on this decision table:

[cols="1,3"]
|===
| `WRITE_NEW`  | Target does not exist - write the file.
| `MERGE`      | Target exists with exactly one valid marker pair - replace only the managed block, leave everything outside untouched.
| `OVERWRITE`  | Target exists - replace wholesale (explicit action).
| `SKIP`       | Target exists with no markers (hand-authored) - never clobbered.
| `CORRUPTED`  | Markers are broken (reversed / duplicated / half-missing) - requires explicit user override; never auto-merged.
| `UNREADABLE` | Target exists but cannot be read (permissions, lock, I/O error) - never written.
|===

== Stability tiers

*Stable* - safe to integrate against; changes here follow the versioning policy
below:

* File paths and directory layout (`AGENTS.md`, `.ai/*`, `.claude/`, `.cursor/`,
  `.windsurf/`, `.launchpad/*`).
* The `launchpad:provenance` and `launchpad:managed:start/end` marker tokens.
* The `FileKind` discriminator and the write-action decision table.
* `.ai/*` heading anchors (explicit `{#slug}` anchors derived from each record's
  stable `id`, so chunk anchors survive title edits).
* The deterministic sidecars under `.launchpad/`, each governed by its own
  `schemaVersion`.

*Experimental* - may change run-to-run; do not snapshot-test:

* AI-synthesized prose, chiefly the `AGENTS.md` Intro. Its wording depends on
  the local model and is not stable across runs. Integrate against the
  *structure* (section presence and order), not the sentences.

== Versioning policy

* Every machine-readable artifact carries its own version field; bumping it
  signals a breaking change to that artifact's shape. The provenance payload
  uses `schemaVersion`; the deterministic sidecars each use `schemaVersion`
  (see xref:project-model.adoc[] and xref:standards-index.adoc[]).
* `generatedAt` is the only non-deterministic value and is deliberately excluded
  from every content hash, so two runs of an unchanged project differing only by
  timestamp produce identical content.
* Marker tokens and file paths are part of the contract: they change only with a
  documented migration, never silently.

See also xref:architecture.adoc[] for the generation pipeline and
xref:mcp-tools.adoc[] for querying the same model over MCP.
