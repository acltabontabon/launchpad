=== RULES ===

Add rules covering:
- **Pin provider versions**: lock `databricks/databricks` and any cloud provider to specific versions in `versions.tf` / `required_providers`. Never use unbounded `~>` or unset versions for production.
- **Secrets never inline**: secret values come from Databricks secret scopes or remote state (Vault / AWS Secrets Manager / GCP Secret Manager), never inline in Terraform files.
- **State backend is remote**: local `terraform.tfstate` is never committed; backend is configured for shared remote state with locking.
- **Idempotent applies**: every `terraform apply` from a clean checkout must converge to the same workspace state; one-shot manual workspace edits do not exist.
- **Plan before merge**: changes that touch `databricks_job` / `databricks_pipeline` / `databricks_cluster` require a `terraform plan` review attached to the PR.
