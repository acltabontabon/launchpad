=== SKILLS ===

Include a skill named "add-kafka-consumer" that:
- Names the actual messaging package from the context
- Defines a typed payload record
- Configures the topic name through @ConfigurationProperties, not @Value
- Includes a DLT (dead-letter topic) handler

=== RULES ===

Add rules covering:
- **Idempotent consumers**: handlers must tolerate redelivery without producing duplicate side-effects.
- **DLT for every listener**: every @KafkaListener has an explicit dead-letter destination, not just default retry.
- **Typed payloads**: deserialize into records, not Maps; reject malformed payloads early.
